# Архитектура SportMax ERP

## Обзор

Система построена как **монорепо** с разделением на несколько независимых приложений, которые общаются через REST API.

```
                    ┌──────────────────────────────────┐
                    │           Интернет               │
                    └──────────────┬───────────────────┘
                                   │
                    ┌──────────────▼───────────────────┐
                    │         Caddy (HTTPS)            │
                    │    reverse proxy + SSL           │
                    └──────┬───────────────┬───────────┘
                           │               │
              ┌────────────▼──┐    ┌───────▼────────────┐
              │  Next.js Web  │    │   NestJS API       │
              │  :3000        │    │   :3001            │
              │  (фронтенд)   │◄──►│   /api/*           │
              └───────────────┘    └──────┬─────────────┘
                                          │
                           ┌──────────────┼──────────────┐
                           │              │              │
                  ┌────────▼──────┐ ┌────▼─────┐ ┌──────▼───┐
                  │  PostgreSQL   │ │  Redis   │ │  Uploads │
                  │  :5432        │ │  :6379   │ │  /uploads│
                  └───────────────┘ └──────────┘ └──────────┘
```

## Компоненты

### 1. Next.js Web (`apps/web`)

**Что это:** Фронтенд-приложение, которое видят пользователи в браузере.

**Как работает:**
- Использует **App Router** (папка `app/`) — современный способ роутинга в Next.js 15
- Серверные компоненты (Server Components) рендерятся на сервере — быстро загружаются
- Клиентские компоненты (Client Components, `'use client'`) — интерактивные части
- Все данные берёт из NestJS API через `lib/api.ts`

**Структура папок:**
```
apps/web/
├── app/
│   ├── (auth)/login/        # Страница входа
│   └── (dashboard)/         # Весь интерфейс после входа
│       ├── layout.tsx        # Общий layout (Sidebar + Header)
│       ├── page.tsx          # Дашборд /
│       ├── clients/          # /clients
│       ├── crm/              # /crm
│       ├── reception/        # /reception
│       └── ...
├── components/
│   ├── layout/              # Sidebar, Header, Providers
│   └── ui/                  # Переиспользуемые компоненты
└── lib/
    ├── api.ts               # Клиент для запросов к API
    └── utils.ts             # Вспомогательные функции
```

---

### 2. NestJS API (`apps/api`)

**Что это:** Бэкенд-сервер, обрабатывает все запросы, работает с базой данных.

**Как работает:**
- Принимает HTTP-запросы на `/api/v1/*`
- Проверяет JWT-токен (авторизация)
- Проверяет роль пользователя (RBAC)
- Выполняет бизнес-логику
- Читает/пишет в PostgreSQL через Prisma
- Возвращает JSON

**Структура папок:**
```
apps/api/src/
├── main.ts                  # Точка входа, настройка сервера
├── app.module.ts            # Главный модуль, подключает все остальные
├── prisma/                  # Сервис для работы с БД
│   ├── prisma.module.ts
│   └── prisma.service.ts
├── common/                  # Общие инструменты
│   ├── decorators/          # @Roles(), @CurrentUser()
│   ├── guards/              # JwtAuthGuard, RolesGuard
│   └── filters/             # Обработка ошибок
└── modules/                 # Бизнес-модули
    ├── auth/                # Вход в систему
    ├── clients/             # Клиенты
    ├── crm/                 # Воронки и сделки
    ├── visits/              # Посещения
    ├── rfid/                # RFID-карты и доступ
    ├── loyalty/             # Программа лояльности
    ├── schedule/            # Расписание занятий
    ├── finance/             # Платежи и финансы
    └── mango/               # Интеграция с телефонией
```

**Каждый модуль состоит из:**
```
module/
├── module.module.ts     # Регистрирует контроллер и сервис
├── module.controller.ts # Принимает HTTP-запросы, возвращает ответы
├── module.service.ts    # Бизнес-логика, работа с БД
└── dto/                 # Data Transfer Objects (схемы данных)
```

---

### 3. PostgreSQL (база данных)

**Что хранит:** Все данные системы — клиенты, сотрудники, абонементы, посещения, сделки, платежи и т.д.

**Как доступна:** Через Prisma ORM в NestJS API. Напрямую из Next.js НЕ доступна.

**Схема:** Описана в `apps/api/prisma/schema.prisma`

---

### 4. Redis (кэш и очереди)

**Что хранит:**
- Кэш частых запросов (список клиентов, статистика)
- Очереди задач BullMQ (отправка уведомлений, отчёты)
- Сессии (опционально)

---

### 5. packages/shared (общие типы)

**Что это:** TypeScript-типы и енумы, которые используются и в API, и в Web.

**Зачем:** Чтобы не дублировать определения типов. Если API возвращает `IClient`, то в Web тоже используется этот же тип.

## Поток данных (пример: список клиентов)

```
Пользователь открывает /clients
        │
        ▼
Next.js рендерит страницу (app/(dashboard)/clients/page.tsx)
        │
        ▼
TanStack Query делает GET http://localhost:3001/api/clients
        │
        ▼
NestJS ClientsController.findAll() получает запрос
        │
        ▼
JwtAuthGuard проверяет токен → OK
        │
        ▼
ClientsService.findAll() выполняет Prisma-запрос
        │
        ▼
PostgreSQL возвращает данные
        │
        ▼
JSON ответ → TanStack Query кэширует → React рендерит таблицу
```

## Аутентификация и авторизация

### JWT-токен
1. Пользователь вводит email + пароль
2. `POST /api/auth/login` → сервер проверяет пароль (bcrypt)
3. Сервер создаёт JWT-токен со сроком 7 дней
4. Токен сохраняется в localStorage
5. Каждый запрос отправляет `Authorization: Bearer <token>`
6. `JwtAuthGuard` на каждом защищённом эндпоинте проверяет токен

### RBAC (роли)
```typescript
// Защита по роли прямо на методе контроллера:
@Get('stats')
@Roles('ADMIN', 'DIRECTOR')  // только эти роли
@UseGuards(JwtAuthGuard, RolesGuard)
getStats() { ... }
```

## Офлайн-режим (локальный сервер)

Для работы турникетов без интернета предусмотрена концепция **SportMax Agent** — локального приложения на компьютере клуба:

```
Турникет → SportMax Agent (локально)
               │
               ├─── Локальная БД (SQLite/PostgreSQL local)
               │    Хранит: активные RFID, абонементы
               │
               └─── Синхронизация с облаком (при наличии интернета)
```
